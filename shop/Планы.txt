Кнопки Массового апдейта в backend
1. Назначение рекомендуемых товаров по категориям, для каждой категории назначаем категории рекомендуемых.
2. Назначение Меток SEO по категориям.
3. Обработка брендов: по списку брендов ищим в $product->name совпадение.

Нужные аттрибуты
для кабеля - сечение, жилы, тип (ВВГ-нг, ПВС, КГ, СИП, ПУГНП, ПУНП, ШВВП, АПВ, ПВ-1, ПВ-3, АВВГ, РКГМ, АВБбШв)


Загрузка продуктов:
а) при изменении цены, если цена стала меньше, то price_new -> price_old

    + Отдельный пункт в backend:
    1) выбираем каталог
    2) выбираем аттрибут
    3) пишем регулярку (выбираем из списка по code1C каталога, а не по его id)
    4) find
    выдает список по всем продуктам:
    товар     что нашел    [x] принять
    5) accept



б) Атрибуты для кабеля: кол-во жил х сечение:
    ([0-9]{1})[x|х]([0-9]{1,3}[,]?[0-9]*)
    если групп больше одной (то жилы складываем, например 3+1, сечение по максимальному)
в) аттрибуты для освещения Мощность, Цвет
    ([0-9]+)(Вт|W) /is - берем только первый элемент (индекс 1)
    ([0-9]{4})[K|К]+?/is
г) для автоматов: Ампераж, кол-во модулей
    ([0-9]{1,3})[A|А]
    ([1234]{1})[P|Р]
д) Для крепежа - диаметр, длина
    по разделам свои данные

Расширенный поиск
1. По умолчанию, аттрибуты, отключены.
2. Подключаются по Ajax (Pjax) только при выборе Категории
(По Id Категории выбираем все product_id, в Value по product_id выбираем все characteristic_id, которые и подгружаем)
